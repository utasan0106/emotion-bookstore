# 天気連動hotfix — 実ブラウザ手動確認手順

自動テスト環境（jsdom）では実ブラウザのCSP適用・実API通信・実レイアウトを確認できないため、
以下の手順での確認をお願いいたします。所要時間は5分程度です。

## 準備：ローカルHTTPサーバーで開く

```
cd emotion-bookstore-main
python3 -m http.server 8000
```

ブラウザ（Chrome推奨）で http://localhost:8000/ を開きます。

## 確認1：CSP適用下での実API通信

1. DevTools（F12）→ Console タブを開いておく
2. 「一冊つくる」等で入店 → 店内メニューを開く
3. 「天気連動：切」を押してオンにする → 「主要地域から選ぶ」で任意の都市を選択
4. **確認A**：DevTools → Network タブで `api.open-meteo.com` へのリクエストが **200** で成功していること
5. **確認B**：Console に `Refused to connect ... Content Security Policy` などの**CSP違反が出ていない**こと
6. **確認C**：店内上部に「東京・晴れ」などのバッジが表示されること（天気状態により文言は変わります）

## 確認2：現在地から設定

1. 「現在地から設定」を押す → このタイミングで初めて位置情報の許可ダイアログが出ること
   （ページを開いただけでは出ないこと）
2. 許可した場合：ステータスに「現在地周辺」と表示され、緯度・経度の数値が画面に出ないこと
3. 拒否した場合：エラーにならず通常表示のまま使えること

## 確認3：390px幅の上部バー

1. DevTools → デバイスツールバー（Cmd+Shift+M）→ 幅390pxに設定
2. 店内上部の「店名・日付・天気バッジ・メニューボタン」が重ならないこと
3. 言語切替（JP/EN）後の長い英語表示でも横スクロールが発生しないこと
   （バッジ・日付は省略記号「…」で切り詰められる設計です）

## 確認4：ポップオーバーの操作

1. 天気バッジをクリック → 地域・天気・最終更新の3行だけのポップオーバーが開くこと
2. Escapeキーで閉じること
3. もう一度開いてから、ポップオーバーの外側をクリックして閉じること

## 確認5：言語切替

1. 天気バッジ表示中に JP/EN を切り替える
2. 「東京・雨」⇔「Tokyo · Rain」のように、再読み込みなしで即座に切り替わること
3. Network タブで、言語切替により `api.open-meteo.com` への**追加リクエストが発生しない**こと

## 確認6：帰属表示

1. 店内メニューの天気設定の下に「天気データ：Open-Meteo（CC BY 4.0）」
   （英語モード：Weather data by Open-Meteo (CC BY 4.0)）が表示され、
   2つのリンクがそれぞれ open-meteo.com と creativecommons.org へ飛ぶこと
