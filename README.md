# みんなの感情書店 — 公開・運用メモ

## 現在の仕様

「みんなの感情書店」は、登録やログインをせず、気持ちを書いて一冊の本として残せる静的ウェブアプリです。

- 利用者が入力した本文、題名、写真、本棚データは、原則として端末内のIndexedDB / LocalStorageへ保存します。
- 下書きは最後の保存から7日間保持します。製本成功後に削除し、製本失敗時は保持します。
- 店主まなはAIチャットではありません。利用者本文を分析・診断・自動分類せず、状態に応じた承認済みの固定文を表示します。
- 新しい本は中立内部ID `unfiled` で保存します。製本後、21種類の棚から任意で選ぶか、棚を選ばず本棚へ戻れます。
- 書籍・音楽検索APIへ送るのは運営側が定めた固定検索語だけで、利用者本文は送りません。
- X公式埋め込みはリリース時に停止しています。Xへの手動シェアと公式アカウントへの通常リンクだけを使用します。
- 実行時に生成AI・LLM・外部AI APIは使用しません。

## ファイル構成

- `index.html`：画面構造、メタ情報、GA4初期設定
- `main.js`：画面制御、保存、製本、棚、本棚、GA4カスタムイベント
- `data.js`：21棚、書籍・音楽・表示用コンテンツ
- `style.css`：画面デザイン
- `guide.html`：はじめての方へ・FAQ・相談窓口
- `privacy.html`：プライバシーポリシー
- `terms.html`：利用規約
- `sitemap.xml`：サイトマップ
- `vercel.json`：Vercelレスポンスヘッダー
- `manifest.json`：最小限のWeb App Manifest
- `sw.js`：Service Worker
- `ogp.png`：OGP画像
- `package.json`：プロジェクト情報
- `README.md`：本書

## 公開方法

全ファイルを同じ構成のままGitHubリポジトリへ配置し、Vercelの静的サイトとして公開します。
サーバー関数、生成AI用APIキー、環境変数は使用しません。ファイルを一部だけアップロードせず、完全ソース一式を配置してください。

## GA4

アプリが明示的に送信するカスタムイベントは次の5種類です。

- `view_landing`
- `start_writing`
- `create_book_success`
- `create_book_error`
- `view_shelf`

イベント名以外の独自パラメータは送りません。本文、題名、棚名、感情名、写真名、書名、曲名、作品ID、エラー内容を送信しません。
公開前にGA4管理画面の拡張計測から「離脱クリック」をOFFにしてください。

## 公開前の主な確認

- スマートフォンで番台、編纂机、製本、棚選択、スキップを確認
- 通常棚の本と`unfiled`の本が混在しても表示が崩れないことを確認
- 下書きの復元、製本成功時の削除、製本失敗時の保持を確認
- バックアップの保存・復元を確認
- プライバシーポリシー、利用規約、guide.htmlへのリンクを確認
- X公式埋め込み通信が発生しないことを確認
- GA4の5イベントと、禁止データが送信されないことを確認

## 変更時の注意

利用者本文を分析・診断・自動分類する処理、本文連動の店主応答、本文連動の棚提案、題名提案、外部AIへの送信を追加しないでください。
店主の言葉は、状態に応じた承認済み固定文として管理してください。
