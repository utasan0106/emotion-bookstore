# みんなの感情書店 — 公開手順（Vercel／サーバー関数なし版）

**このバージョンから、店主AIはすべてブラウザ内で完結するようになりました。** 対話・納品審査・栞のすべてがキーワード照合とテンプレートによるローカル処理で動くため、**外部AI APIの呼び出しは一切なく、APIキーの取得や料金の心配は不要です。**

- `index.html` … アプリ本体（見た目・棚・製本・本棚・演出・店主の頭脳、すべて含む）
- `api/shopkeeper.js` … 過去バージョンで使っていた「本物のAI対話」用のサーバー関数です。**現在のindex.htmlは呼び出していないため削除しても構いません。** 将来、本物の生成AIに戻したくなった場合の参考として残しています
- `package.json` … プロジェクトの基本情報

## 公開手順（GitHub + Vercel）
1. https://github.com で新しいリポジトリを作成
2. `index.html` と `package.json` をアップロード（`api/` フォルダは無くてもそのままでも、どちらでも動きます）
3. https://vercel.com にGitHubでログイン →「Add New...」→「Project」→ リポジトリを「Import」→「Deploy」
4. 環境変数の設定は不要です。数十秒でURLが発行されます

## 店主の頭脳の中身（すべて `index.html` 内）
- **番台の対話**：`KEYWORD_BANK` に登録した言葉（既読・部活・テストなど）にマッチしたら、対応する返答をランダムに表示。ヒットしなければ深い相槌を返します。「死にたい」等の深刻な言葉は最優先で検知し、相談窓口を案内します
- **納品審査**：`localCurate()` が、個人情報らしき文字列・攻撃的な言葉・深刻な訴えを簡易チェックし、棚の自動提案もキーワードの一致数で行います
- **今日の栞**：`localShiori()` が、本棚の傾向に応じたテンプレート文をランダムに選んで表示します

**中身を増やしたいときは、この3つの定数・関数がある場所を探して項目を足すだけ**でバリエーションが増えます（コメントに目印を書いてあります）。

## 収益化の設定（重要）
- **Amazonアソシエイト**：`index.html` 内の `const AMAZON_ASSOCIATE_ID = 'your_id-22';` を自分のIDに書き換えると、**全ての本のAmazonリンクと、曲のAmazon Musicリンクに自動でトラッキングタグが付きます**。書き換えるまではタグなしの通常リンクとして動きます（リンク切れの心配はありません）
- 楽天は現状、通常の検索リンクです。楽天アフィリエイトを利用する場合は楽天側で発行したリンク形式への変更が必要です

## Xの実投稿を棚に表示する（任意）
`const PINNED_TWEETS = {...}` に、その棚の気持ちを代弁している実在の投稿URLを貼ると、棚を開いた時にXの**公式埋め込み**で表示されます（規約に沿った正式な引用方法です）。
例：`moya:['https://x.com/ユーザー名/status/1234567890'],`
※未ログインの閲覧者には埋め込みが表示されない場合がありますが、その時は「Xで見る」リンクに自動で切り替わります。

## Instagramストーリー用の画像保存
「今日の栞」を受け取った後に表示される「📸 ストーリー用の画像で保存」ボタンで、本棚と栞を合成した縦長（9:16・1080×1920px）のPNG画像が端末に保存されます。外部ライブラリ不使用のCanvas直描画なので、オフラインでも確実に動きます。

## ホーム画面アプリ化（PWA風）
スマホで「ホーム画面に追加」すると、URLバーのない全画面表示で起動します。アイコンは現在SVGの簡易版です。公開時は180×180pxのPNGを用意し、`<link rel="apple-touch-icon" href="...">` を`<head>`に足すとより綺麗になります。

## 音楽との連携について
歌詞や動画そのものをアプリ内に保存すると著作権上のリスクがあるため、今回は「その気分に合う曲をYouTubeで探す」というリンク（YouTubeの公式検索結果ページを新しいタブで開く）に留めています。特定の動画を画面内に埋め込みたい場合は、実在する動画IDを一つずつ確認しながら追加する必要があります（存在しない・削除された動画IDを埋め込むと表示が壊れるため）。

## 参考にしたサイトについて
UI・文章のトーン作りにあたり、以下のサイトを実際に確認し、良いと感じた要素（一言フック付きの本紹介、書棚という比喩、感謝ソングの選曲傾向など）を参考にしました。ただし、いずれのサイトの文章・画像も直接転載はしておらず、着想を得た上で本サービス用に書き起こしています。
- Book Bang（bookbang.jp）：新聞・出版社の書評まとめサイト。レビュアーと媒体名を添える誠実な出典表示のスタイル
- ブクログ（booklog.jp）：本棚というメタファーの使い方、一言フック付きの本紹介
- AWA「16音楽キャラ診断」：気分から音楽を提案するという企画の考え方
- その他ご共有いただいたサイトのうち、実際にアクセスできたものを中心に参照しています

## 仕様上の注意
- 本棚・栞のデータは各訪問者のブラウザ内にのみ保存されます（フッターにも明記済み）。機種変更やブラウザのデータ削除で消えます
- 店主のビジュアルは権利上、実在の人物を模したものにはできないため、オリジナルの人物像（後ろ姿のシルエット）にしています
- 名言は「店主のオリジナルの言葉」と「著作権の切れた文学（太宰治・夏目漱石など、出典明記）」のみを使用しています。現代の漫画・歌詞は著作権リスクが高いため意図的に含めていません
